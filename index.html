HTML

HTML:
Hypertext Markup Language is the backbone of the content on the web. When you view a 'web' page through your browser, the browser is rendering the HTML in a manner that is visually accommodating to people, and hopefully, in an attractive and organized manner. The hypertext is the content and the 'markup' tells the browser how to organize and display the information.

HTML Tags:
<tags> comprise the basic grammar and syntax of HTML. There are opening <tags> and closing </tags> that will act to organize and display content in specific manners; and, everything between the opening and closing tag is called an element. Most, but not all, <tags> will have a closing tag. A few tags such as <br> or <img>do not have closing tags, and are called 'void' tags.

Innline vs Block Tags:
Another distinction between tags is whether or not they are considered to 'inline' or 'block' tags. Inline and block tags organize the white space within a document in different ways. A block tag imparts a dimensionality to the element that is interpreted as a box or block, whereas an inline, according to the Mozilla Development Network: "An inline element occupies only the space bounded by the tags that define the inline element." Examples of inline tag is the <br>
that wraps the text. And the <span> tag that may be used to alternatively style a block of text.

An example of a block tag is the <p> or paragraph tag.

Attributes:
Elements may have attributes; and, attributes will have 'values' that may modify the element. Some elements are required to have attributes like <img> and <a>. So, in order to show the image of, and provide a link to a tribute article to the very first computer I ever used (the TRS-80), I would need to use the attributes of 'href' to provide the link and 'src' to provide the path to the image. Lesson 2: Basics of the HTML Structure

Structure of a Web Page

A web page is put together, in general, with HTML, which provides the structure and foundation of the the page; CSS, which provides the style; and JavaScript, which will 'fancify' your web page by allowing certin functionallity. A useful way to understand a web page's coding and organization is to 'boxify' the layout by visualizing or drawing boxes around the major content compononents. Each box will represent a different section and that leads us to using the 'div' tag, which is the basic tag used to define the boxes/structure in a page.

Document Object Model (DOM)

The 'DOM' ( da DOM DOM) represents the structure of the html and has a tree like structure tht shows the hierarchal relations of the elements within a page. The DOM can either be built manually or you can be smart and view it in developer tools. The DOM shows the relationship of the structural components and how different components or elements are nested within the boxes; and also how different boxes or 'divs' are nested within other boxes.

Tools for the Front-end Developer

Two tools are HTML Editors and the Developer Tools within a browser. HTML editors are specically designed to help write copy and will have features that will help minimize the repetivie nature of having to type the same thing over and over again. For example, I generally use the CoffeCup Editor. The other really useful tool is the 'Developer Tools' feature within the browser. The developer tools not only allow you to see the html, but alsow to view the style, the box dimensions, and very importantly, allow you to alter tags and elements to see what the result would be. The developer tool also provides you with the DOM. Other useful tools include HTML and CSS code validators. These are awesome in that they will allow you more easily trouble-shoot any mistakes that you have made in your code.

Lesson 3: Basics of Style and CSS

What is CSS?

CSS is short for Cascading Style Sheets. CSS allows you to impart style or 'purtify' your web page. 'Cascading' means that CSS rules follow rules of inheritance in that descendant elements will inherit the style properties of the ancestors unles established otherwise. Within CSS, the most specific rule is applied across the document. For example, if I say that all'p' elements will have a blue font color, all 'p' elements will have a blue font-color, unless I more specfically provide a class or id value to a 'p' element and have set it up for that class or id to produce a red font.

How May CSS Be Applied

CSS may be applied in one of three ways:

  1. Inline to an element although, this should be avoided at all costs it requires repeqting code. With the notable except of writing HTML emails. This is a whole 'nother class, though...
  2. Within the HTML document 'head' section within 'style' tags. Again, avoid this for web-pages as you want to SEPARATE YOUR CONCERNS
  3. Within a linked style sheet. The recommended and web-page approved manner.

One of the biggest advantages of using CSS is that it allows you avoid having to repeat the same instructions for like elements.

Box Model

When understanding style for web-pages, it is important to grasp the concept of the 'Box Model'. The Box Model simply delineates the dimensions and components of a box.

The dimensions of a box are determined by the:

  1. The margin which defines the 'whitespace' outside of a box. Although the margin is not used in the calculation.
  2. The border or the edge around the box.
  3. The padding within the box, or the 'buffer' space around the conent within a box.
  4. The content width and height within the box.

Now this can be all very confusing when trying to figure and determine boxes. So there a couple of things that you can do to make your life easier when formatting the boxes:

  1. Set sizes of the box in terms of percentages, e.g. 70%.
  2. Set box-sizing attribute to border-box across all the elements
